## O que são?

Requisitos funcionais descrevem **o que o sistema deve fazer** — os serviços, comportamentos e interações entre o sistema e seus usuários ou entidades externas.

**Formato padrão:**
> `[quem]` + `[deve poder fazer algo]` + `[com qual entidade/dado]`

## 📌 Lista de Requisitos Funcionais

### Autenticação e Usuários

| ID | Requisito |
|---|---|
| RF01 | O sistema deve permitir que o usuário realize cadastro com e-mail e senha. |
| RF02 | O sistema deve permitir que o usuário faça login com suas credenciais. |
| RF03 | O sistema deve permitir que o administrador gerencie contas de usuários. |

### Funcionalidades Principais

| ID | Requisito |
|---|---|
| RF04 | O sistema deve permitir que o usuário visualize a listagem de itens disponíveis. |
| RF05 | O sistema deve permitir que o usuário realize busca por nome ou categoria. |
| RF06 | O sistema deve permitir que o usuário acesse os detalhes de um item específico. |

### Relatórios e Administração

| ID | Requisito |
|---|---|
| RF07 | O sistema deve permitir que o administrador gere relatórios filtrados por período. |
| RF08 | O sistema deve permitir que o administrador visualize logs de acesso. |

---

## ✅ Critérios de Qualidade

Para que um requisito funcional seja válido, ele deve ser:

- **Completo:** todos os serviços exigidos pelo usuário devem ser definidos.
- **Consistente:** sem definições contraditórias entre requisitos.
- **Mensurável:** deve ser possível testar se o requisito foi atendido.
